# COLLECTORS — Scheduler & Memory Collection Design

Research deliverable for mission `cpp-sched-mem-profiler`. Every collector maps to a real Linux
kernel interface and reports a capability state (`READY`, `DEGRADED`, `SKIP`, `REFUSE`) so the
pipeline can fail closed when a feature is unavailable.

## 1. Capability model

Each collector implements:

```cpp
struct Capability { std::string name; CapState state; std::string reason; };
class ICollector {
public:
  virtual Capability probe() = 0;          // read-only, never mutates
  virtual bool start(RingSink&) = 0;       // begins emission
  virtual void stop() = 0;
};
```

`probe()` is pure observation. If the kernel interface is missing or permission is denied, the
collector returns `SKIP`/`REFUSE` with a reason and emits no events. This is the C++ form of the
Zig project's fail-closed behavior.

## 2. Scheduler collectors

| Signal | Kernel interface | Notes |
| --- | --- | --- |
| context switches | tracepoint `sched:sched_switch` via `perf_event_open(PERF_TYPE_TRACEPOINT)` or BPF `raw_tracepoint/sched_switch` | prev_comm/prev_pid/prev_state/next_comm/next_pid |
| wakeups | `sched:sched_wakeup`, `sched:sched_waking` | waking task + target cpu; basis for unnecessary-wakeup detection |
| migrations | `sched:sched_migrate_task` | orig_cpu -> dest_cpu |
| run queues | per-CPU `/proc/stat` + `schedstat` (`/proc/schedstat`), or BPF runqlen sampling | occupancy + nr_running |
| CPU utilization | `/proc/stat` per-CPU jiffies, `mpstat`-style deltas | read-only |
| NUMA placement | `/proc/<pid>/numa_maps`, `sched:sched_move_numa`/`sched_swap_numa`, `numa_balancing` sysctl read | current vs. preferred node |
| priority inversion | BPF: track lock owner on `sched_switch` + futex/mutex tracepoints | high-prio waiter blocked on low-prio owner that is preempted |
| lock contention | `lock:contention_begin`/`contention_end` tracepoints, or BPF on `__mutex_lock_slowpath` | wait time per lock site |

sched_ext note: the host exposes `/sys/kernel/sched_ext` (`state`, `switch_all`, `hotplug_seq`,
`enable_seq`, `nr_rejected`) and `CONFIG_SCHED_CLASS_EXT=y` with BTF at `/sys/kernel/btf/vmlinux`.
A CO-RE BPF scheduler can be loaded **only inside the disposable VM lab** and only through the
`safety` gate; on the host the sched_ext collector is read-only (reads sysfs state) and never loads.

## 3. Memory collectors

| Signal | Kernel interface | Notes |
| --- | --- | --- |
| page faults | `perf_event_open` `PERF_TYPE_SOFTWARE / PERF_COUNT_SW_PAGE_FAULTS[,MIN,MAJ]` | per-thread counts + optional PEBS sample |
| TLB misses | `PERF_TYPE_HW_CACHE` dTLB-load-misses, or AMD IBS `ibs_op` tags | PMU; needs `perf_event_paranoid<=1` or `CAP_PERFMON` |
| cache misses | `PERF_COUNT_HW_CACHE_MISSES` / LLC-load-misses; AMD IBS | per-region hotspot when sampled with callchains |
| huge pages | `/sys/kernel/mm/hugepages/*`, `/proc/meminfo` (HugePages_*) | read-only poll |
| NUMA locality | `/proc/<pid>/numa_maps`, `migrate_pages`/`mbind` observation, `numa_hint_faults` from `/proc/vmstat` | local vs. remote allocation |
| allocator fragmentation | `/proc/buddyinfo`, `/proc/pagetypeinfo`, slabinfo | read-only |
| malloc hotspots | optional `LD_PRELOAD` shim `libxsprof_alloc.so` intercepting malloc/free/mmap with backtrace | opt-in; writes to a ringbuf only, never changes app behavior beyond interposition |

PMU reality on this host (source: `/sys/bus/event_source/devices`): `cpu`, `software`, `ibs_fetch`,
`ibs_op`, `msr`, `power`, `power_core`, `kprobe`, `uprobe`, `tracepoint`, `breakpoint`. AMD IBS gives
precise memory-access profiling; the memory collector prefers IBS when `ibs_op` is present and
falls back to generic HW_CACHE events otherwise.

## 4. perf_event_open wrapper (RAII)

```cpp
class PerfEvent {
  int fd = -1;
public:
  static tl::expected<PerfEvent, Err> open(perf_event_attr&, pid_t pid, int cpu, int group);
  // mmap ring page for samples; epoll-driven read in the collector thread
};
```

All FDs are RAII-owned; `paranoid=2`/permission errors map to `CapState::SKIP` with the exact errno
string in `reason`. No collector retries with elevated privileges; elevation is an explicit operator
decision handled outside the framework.

## 5. BPF collector (CO-RE)

- Objects compiled with `clang -target bpf -O2 -g` using the host `vmlinux.h` generated by `bpftool
  btf dump file /sys/kernel/btf/vmlinux format c` (CO-RE, no kernel headers required at runtime).
- Loaded via libbpf skeleton; ringbuf events drained into the pipeline.
- Load is gated: `bpf` collector `probe()` returns `SKIP` on the host unless an explicit
  `--allow-bpf-load` flag plus the VM-lab marker are present (mirrors the Zig lab-only rule).

## 6. Privacy + safety filtering at the collector edge

- Runtime samples never carry argv, environment, or secrets. Comm fields are bounded (16 bytes) and
  optional; any field matching `secret|key|token|password|credential` patterns is redacted before it
  reaches the pipeline.
- Collection is read-only by default. Any path that would write scheduler/memory policy is refused
  and recorded as an `incident`/`refusal` event with `host_mutation=false`.

## 7. Evidence vs. inference

Grounded: tracepoint names, `perf_event_open` types, `/proc` and `/sys` paths, and the PMU list above
were probed on the target host (see `SAFETY.md` for the exact permission posture). Assumption
(labeled): the precise field layout of each tracepoint is read at runtime from
`/sys/kernel/tracing/events/<sys>/<name>/format` when readable, otherwise from a committed BTF/CO-RE
definition; we do not hardcode struct offsets across kernel versions.
